# GoldDIGR data deposit

This directory accompanies the GoldDIGR manuscript and its Supporting
Information. Each file is cited by SI section number. The deposit is
organized into four tiers by use case:

| Tier | Contents | Total size |
|---|---|---:|
| 1. Master tables | DOI ↔ method, converged reactions, full energies, OS table | ~343 MB |
| 2. Convergence analyses | Functional × element-class crosstabs (Section S15) | ~1.4 MB |
| 3. Subsets / audits | Per-element subsets, noble-gas audit (Sections S14, lanthanide rebuttal) | ~125 KB |
| 4. Charge/spin bundle | Author-labeled charge/multiplicity ground truth (Section S3) | 4.3 MB compressed / 98 MB unpacked |

Per-reaction files (`input.xyz`, `finished_first.xyz`, ..., BEM JSONs)
ship inside the per-reaction archives and are not separately listed here.

**Schema notes for `paper_methods.csv`** (48,318 DOIs, see SI §S15):

```
doi              e.g. 10.1021/jacs.5c00077
functionals      "B3LYP | M06" (|-separated canonical list, or empty)
basis_sets       "6-31G(d) | def2-TZVP" (|-separated)
softwares        "Gaussian 09 | ORCA" (|-separated)
elements         space-separated element symbols, e.g. "C H N O Fe"
```

24,035 (49.7%) of rows carry a non-empty `functionals` value. The
~50% empty rate aggregates three indistinguishable causes (the
methods-text keyword pre-filter missing the source's Methods section,
the LLM extractor omitting the field, or the source SI not stating it);
the 262,706-reaction control bin discussed in SI §S15 is built from
these rows.

**Schema notes for `All-reactions.txt`**:

```
# Sigma Bond Changes
# Format: <DOI>/<SI_stem>/<reaction>    <bond_change1>;<bond_change2>;...
# Each bond change: <Atom1>-<Atom2>:<before>><after>
# Bond order values: n = no bond, 0 = dative bond, 1/2/3 = single/double/triple bond
```

472,120 unique reaction keys.

## Scripts

All scripts are in `scripts/`. Organized by data-generation stage:

```
scripts/
├── extraction/
│   ├── detect_cc_metadata.py             # 22-pattern keyword pre-filter (Stage 1 of comp-details extractor)
│   ├── llm_extract_comp_details_good.py  # Stage 2 — OpenAI gpt-4o-mini call with 8-field JSON schema
│   ├── copy_comp_details.py              # Stage 3 — per-publisher collator
│   └── extract_comp_details.py           # Stage 4 — per-publisher sanity CSV (not used downstream)
├── methods/
│   ├── extract_methods.py                # comp-details JSON walk + canonicalization tables
│   ├── extract_elements.py               # per-paper element sets from XYZ leaf dirs
│   ├── element_correlations.py           # joins methods × elements → paper_methods.csv
│   ├── xtb_irc_convergence.py            # convergence joins per functional / basis / class
│   ├── plot_histograms.py                # top-20 bar charts for SI summary panels
│   ├── plot_convergence_heatmap.py       # original heatmap renderer
│   └── regenerate_heatmap_bigfont.py     # SI Figure S3 (larger fonts)
├── energies/
│   ├── extract_energies.py               # per-zip parser: pulls DFT FINAL SINGLE POINT + xTB IRC barriers
│   ├── extract_energies.sbatch           # SLURM submission, single node
│   └── extract_energies_array.sbatch     # SLURM array submission, 200 chunks × 5k zips
├── os/
│   ├── build_tm_os_matrix.py             # per-archive OS rollup for Figure 2b
│   ├── draw_tm_os_radial_dials_fullcircle.py  # Figure 2b radial dial renderer
│   ├── audit_metal_os.py                 # per-metal OS-distribution audit (S13)
│   └── audit_iridium_os.py               # focused Ir audit (S13)
├── elements/
│   ├── sum_histograms.py                 # aggregates per-shard element histograms (Figure 2a)
│   ├── proportion_stripe.py              # element-group proportion stripe panel
│   └── element_heatmap_grouped.py        # group-colored PT heatmap (Figure 2a)
├── archive_management/
│   ├── patch_xtb_keep_worker.py          # adds back xTB-scan/wbo_timeseries + bem_snapshots after slimming
│   ├── patch_xtb_keep.sbatch
│   ├── trim_dataset_worker.py            # archive slimming (3.7 TB → 818 GB)
│   └── trim_dataset.sbatch
└── charge_spin/                          # extracted from charge_spin_bundle_2026-05-29.zip
    ├── extract_charge_spin.py
    ├── extract_charge_spin_parallel.py
    ├── submit_charge_spin.sge
    └── parity_corrected_coverage.py
```

### Re-running from scratch

```bash
# 1. Comp-details extraction (one-time, requires OpenAI API key)
python3 scripts/extraction/detect_cc_metadata.py <pdf-text-tree>
python3 scripts/extraction/llm_extract_comp_details_good.py <flagged-tree>

# 2. Canonicalization + master table assembly
python3 scripts/methods/extract_methods.py
python3 scripts/methods/extract_elements.py
python3 scripts/methods/element_correlations.py
# → paper_methods.csv

# 3. Convergence join
python3 scripts/methods/xtb_irc_convergence.py
# → xtb_irc_convergence_*.csv

# 4. Figure S3 heatmap
python3 scripts/methods/regenerate_heatmap_bigfont.py

# 5. Charge/spin (parallel over 2.3M xyz files, ~34 min on 16 cores)
qsub scripts/charge_spin/submit_charge_spin.sge
python3 scripts/charge_spin/parity_corrected_coverage.py

# 6. Figure 2 panels
python3 scripts/elements/sum_histograms.py
python3 scripts/elements/element_heatmap_grouped.py
python3 scripts/os/build_tm_os_matrix.py
python3 scripts/os/draw_tm_os_radial_dials_fullcircle.py
```

Runtimes: methods → ~minutes; elements → ~minutes; convergence join →
seconds; energies extraction (per-zip) → ~hours per 1 M zips on a 16-core
SLURM job array; charge/spin scan → ~34 min on 16 cores.

## Upstream provenance (files not generated in this deposit)

Several files in Tier 1 are inputs to the analyses here but are
themselves produced upstream by the GoldDIGR core workflow:

- `All-reactions.txt` — produced by the YARP bond-change inference
  step on each archive's BEM trajectory. The patched YARP routine is
  `yarp/find_lewis.py` in the
  [GoldDIGR](https://github.com/Savoie-Research-Group/GoldDIGR)
  repository.
- `transition_metal_oxidation_states.csv` — produced by the OS
  extraction routine described in SI §S13; the routine code is in
  the same `find_lewis.py` (patched 2026-05; see SI §S13 for the
  one-line `el_metals` patch and the three-line non-negativity guard).
- `yyang42_validation_issues_final_result_doi_translated.cleaned.tsv`
  — the pre-validation reject audit referenced in §S4 and docx Q1.
  Produced by the pre-validation pipeline in the GoldDIGR repository.
- Per-reaction archives (`*.tar.zst` or `*.zip`) — the full corpus
  with `input.xyz`, `finished_*.xyz`, `ts_final_geometry.xyz`, BEM
  snapshots, etc. Archived at Zenodo
  `10.5281/zenodo.17503279`.
